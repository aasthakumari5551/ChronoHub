# Requirements Documentation

This folder contains all project requirements and specifications for the ChronoHub Employee Leave Management System.

## Documents

- **Employee_Leave_Management_Assignment_v2.md** - Complete project requirements, tech stack, functional requirements, and grading rubrics

## Project Overview

ChronoHub is a full-stack Employee Leave Management System that simulates a real company HR workflow. The application allows employees to request leave, managers to approve or reject requests, and admins to manage users and permissions.

## Quick Reference

### Mandatory Tech Stack
- Frontend: React.js with Tailwind CSS
- Routing: React Router
- State Management: Context API
- Backend: Node.js with Express.js
- Database: MongoDB
- Authentication: JWT
- Authorization: Role-Based Access Control (Admin, Manager, Employee)

### Core Features
- User Authentication (Login/Register) with JWT
- Role-Based Dashboards
- Apply Leave Form
- Leave Approval Workflow
- Leave Status Tracking
- Protected Routes
- Backend Authorization Middleware
- Global Auth State using Context API
